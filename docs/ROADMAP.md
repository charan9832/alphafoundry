# Roadmap

## Phase 0: Clean product restart

- Fresh standalone repo.
- Pi dependencies and adapter facade.
- Repo-local workflow docs.
- Safety rules and tests.

## Phase 1: Product shell

- `alphafoundry` CLI.
- Onboarding with provider/model/base URL/API-key-env/workspace.
- Doctor/readiness checks.
- Chat entrypoint.
- Mock provider for offline tests.

## Phase 2: Agent runtime and typed tools

- Pi-backed real provider runtime.
- Tool-call loop through registry.
- Streaming events.
- Tool observations with metadata.

## Phase 3: Deterministic finance engine

- Python bridge.
- Data provenance.
- Backtests with costs/slippage.
- Validation checks.
- Report artifacts.

## Phase 4: Full research workflow

- Natural chat: idea -> assumptions -> data -> strategy -> backtest -> validate -> report -> memory.
- Human approval gates for long/costly/destructive actions.

## Phase 5: Product polish

- Rich TUI.
- Charts.
- Workspaces/projects.
- Packaging and installer.
- Optional MCP/export APIs.
